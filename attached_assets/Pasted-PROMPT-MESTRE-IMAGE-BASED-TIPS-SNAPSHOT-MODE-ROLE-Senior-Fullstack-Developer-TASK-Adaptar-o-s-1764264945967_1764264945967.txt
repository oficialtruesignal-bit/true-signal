PROMPT MESTRE: IMAGE-BASED TIPS (SNAPSHOT MODE)
ROLE: Senior Fullstack Developer.

TASK: Adaptar o sistema para suportar Tips Baseadas em Imagem (Prints). OBJETIVO: Permitir que o Admin faça upload de um print (screenshot) do bilhete. Se houver imagem, o Card deve exibir a imagem em destaque em vez de texto.

1. BANCO DE DADOS E STORAGE (SUPABASE)
Precisamos de um lugar para guardar as fotos.

Storage: Crie um Bucket no Supabase chamado tip-images (público).

Tabela: Adicione a coluna image_url na tabela tips.

2. ADMIN: UPLOAD DE IMAGEM (CreateTipForm.tsx)
Adicione um campo de Upload de Arquivo no formulário.

Lógica:

O Admin seleciona o arquivo.

O sistema faz upload para o bucket tip-images.

A URL retornada é salva no banco.

Campos Obrigatórios:

Se tiver Imagem, os campos "Time Casa", "Time Fora" e "Mercado" tornam-se Opcionais (ou podem ser preenchidos genericamente como "Bilhete Pronto").

ODD: Continua Obrigatório (para o cálculo da banca).

Link da Bet: Continua Obrigatório (para o botão de ação).

3. O CARD DE IMAGEM (SignalCard.tsx)
Altere a lógica de renderização do Card.

Lógica Condicional:

Se tip.image_url existir: Renderize o Layout de Imagem.

Se não existir: Renderize o Layout de Texto (o antigo).
